# webapp/data/

`schedule.json` here is the one on-disk schedule file. `app.yaml` serves it
at two URLs:

- `/data/schedule.json` — canonical. Point new app versions here.
- `/schedule.json` — legacy alias, kept only because the iOS 1.0 build
  currently in App Store review still calls it. Remove once no shipped app
  depends on it.

Both handlers set `expiration: "10m"` in `app.yaml`, which bounds how long
an App Engine edge cache can keep serving a stale copy after a new schedule
is published.

Don't hand-edit this file — it's generated by `update_json.py` and checked
by `validate_schedule.py`. See `docs/PUBLISHING.md` ("Schedule JSON URLs"
section) for the full pipeline and rationale.
